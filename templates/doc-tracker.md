# 文档利用追踪

> 每次完成开发任务后运行 `/ecw:knowledge-track` 追加一条记录。积累 15-20 条后做汇总分析。

## 事件类型说明

| 类型 | 含义 | 评分 |
|------|------|------|
| **doc-hit** | 读了文档，直接用上了 | +1 |
| **doc-miss** | 有相关文档但 AI 没找到/没读到 | -1 |
| **doc-redundant** | 读了文档但和代码重复，等于白读 | -0.5 |
| **doc-misleading** | 文档过时/不准确，导致写错 | -2 |
| **code-derived** | 文档没覆盖，AI 费劲从代码推导 | -0.5 |

## 净价值公式

```
文档净价值 = hit×(+1) + miss×(-1) + redundant×(-0.5) + misleading×(-2)
```
