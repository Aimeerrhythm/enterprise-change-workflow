🎯
ecw:risk-classifier
自动将每次变更分级为 P0–P3，路由到对应工作流深度。所有变更的统一入口。
自动：任意变更 / 需求 / Bug
🔀
ecw:domain-collab
并行多域分析。第一轮：各域独立分析；第二轮：跨域冲突协商共识。
跨域变更
❓
ecw:requirements-elicitation
系统化追问到需求清晰为止。写代码之前先把事情搞明白。
单域 P0 / P1
🐛
ecw:systematic-debugging
知识库驱动的根因分析，支持跨域链路追踪。一步步排除，不靠猜。
Bug / 测试失败
📋
ecw:writing-plans
风险感知的方案撰写，注入领域上下文，结构化任务拆解，附带明确验收标准。
分析完成后（P0–P2）
⚔️
ecw:spec-challenge
独立对抗性评审，从另一个角度挑刺，趁还没写代码赶紧暴露问题。
P0 任意 · P1 跨域
🧪
ecw:tdd
先写测试再写代码，覆盖率要求按风险等级区分。
实现前（P0–P2）
🚀
ecw:impl-orchestration
并行 subagent 执行方案 Task。每个 Task 拥有隔离上下文、单任务 spec 评审和自动提交。
4+ 个 Task · P0 / P1
✅
ecw:impl-verify
多维度收敛验证：代码 ↔ 需求 / 领域规则 / 方案决策 / 工程规范，四轴交叉校验。
实现完成后（P0–P2）
📊
ecw:biz-impact-analysis
基于 git diff 分析受影响的业务流程、外部系统和端到端路径。
impl-verify 完成后
🔍
ecw:cross-review
跨文件结构一致性检查——表格、列表、计数、术语在多个关联文档间的一致性。
手动：/ecw:cross-review
🏥
ecw:knowledge-audit
知识库健康检查：内容组成分析、过时引用检测、三标准合规性验证。
手动：/ecw:knowledge-audit
📈

任务后文档利用率追踪——命中、未命中、冗余、误导四类分类，持续优化知识库。
手动：/ecw:knowledge-track
🗺️
ecw:knowledge-repomap
基于 component_types 配置自动生成结构化代码索引，加速上下文感知导航。
手动：/ecw:knowledge-repomap

Agent 架构

7 个 Agent
分层协作

分析层用 Opus 深度推理，执行层用 Sonnet。Agent 之间职责隔离，不能越权调用。

Claude Opus 深度分析层
🔍
biz-impact-analysis
5 步结构化分析：diff → 依赖图 → code 扫描 → 外部系统 → 报告
⚔️
spec-challenge
4 维对抗评审：准确性 / 信息质量 / 边界盲点 / 鲁棒性
🔬
domain-analyst
R1 独立域分析，输出强制 ≤30 行 YAML，防止 context 污染
🤝
domain-negotiator
R2 跨域协商：评估其他域提案对自己域的影响，寻求共识
Claude Sonnet 高效执行层
👷
implementer
单任务实现，含 Fact-Forcing Gate + self-review，100 工具调用 hard limit
🕵️
spec-reviewer
任务级规格合规性审查，独立读代码，不信任 implementer 的 self-report
📐
impl-verifier
4 个维度并行：需求 / 领域规则 / 设计决策 / 工程标准，各自独立校验
FACT-FORCING GATE 编辑前必须引用需求原文

implementer 每次改文件前必须引用 Task 描述中的原句。引用不到就不许改——防止自作主张。

# implementer agent 的内置约束
BEFORE modifying file X, quote the exact line from Task Description requiring this edit.
IF you cannot point to a specific requirement → DO NOT make the edit.
企业级可靠性

上下文不丢
分级越来越准

session-state 保证长任务不怕上下文压缩；校准机制根据历史结果修正风险预测。

🔄 Session 连续性

session-state.md 是工作流的单一真源，新会话通过 SessionStart hook 自动重建完整上下文。

Subagent Ledger：记录每个 subagent 的启动时间、耗时、worktree 路径，跨 session 可追踪进度
Checkpoint 文件体系：requirements-summary / phase2-assessment / domain-collab-report，每阶段完成立即落盘
PreCompact hook：context 压缩前注入恢复指引，防止 long session 中途失忆
标记式增量更新：用 标记，多个 hook 可并发更新不同部分不互相覆盖
🎯 自我校准 Calibration

Phase 3 自动对比预测分级 vs 实际影响，提取 instinct 规则，SessionStart 动态注入高置信度规则。

instincts.md：每条规则附带置信度 0.0–1.0，仅注入 >0.7 的规则，防止低置信度误导
漏报 → 升级规则：预测低于实际时，从关键词提取"考虑升级 1 级"规则，初始置信度 0.5
误报 → 降级规则：预测高于实际时，同理提取降级规则
准确 → 强化规则：相同关键词的规则置信度 +0.1（上限 1.0），越用越精准
PHASE 1 ⚡
快速预测
关键词 + instinct 规则 + 共享资源表，秒级输出初步分级
→
PHASE 2 🔬
精准分级
完整依赖图（§1–§5），确认或升/降级 Phase 1 的预测
→
PHASE 3 📊
校准反馈
实现完成后对比实际影响，更新 calibration-history 和 instincts
→
NEXT SESSION 🧠
规则注入
SessionStart 自动注入置信度 >0.7 的规则，Phase 1 越来越准
